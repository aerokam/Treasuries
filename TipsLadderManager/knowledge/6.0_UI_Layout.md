## Top Card Geometry

The `.form-card` sits below the tab bar, flush (no border-radius on top corners).

- **Width:** `fit-content`, `min-width: 845px` (grows with content)
- **Padding:** `10px 24px 8px`
- **Shadow:** `0 10px 15px -3px rgb(0 0 0 / 0.4)`
- **Position:** Relative (sticky container via `#top-row`)

## Form Card Layout

Three stacked rows inside `.form-card`:

### Row 1 — Inputs (`.card-inputs`)
Flexbox, `gap: 8px 20px`, `align-items: flex-end`.  
Contains: Holdings CSV, DARA, First Year (mode-dep), Last Year (mode-dep), Method, Brackets, Maturity pick (build-only), Pre-ladder int.

### Row 2 — Actions (`.card-inputs`, `margin-top: 8px`)
Contains: Run button (`padding: 5px 18px`), Export CSV (post-run), Export CUSIP/Qty (post-run), Duration links, `#wad-display` (post-run).  
All items `align-self: flex-end`.

**Duration link button labels:** `Gap Dur` (gap/bracket duration calcs popup), `Future 30Y Dur`.

**`#wad-display`** — shown after run (both modes). Displays `Wtd Mod Dur <value>` using `.info-stat` / `.info-stat-val` styling. No unit suffix (modified duration is dimensionless). Hidden until a result is available. Positioned immediately to the right of `#duration-links`.

### Row 3 — Info Strip (`.card-info`)
Single flex row, `gap: 12px`, `align-items: center`, `margin-top: 6px`, `padding-top: 6px`, `border-top: 1px solid rgba(255,255,255,0.12)`.

**Info strip typography:**

## Form Field Arrangement

- **Layout:** Flexbox with `gap: 8px 20px`.
- **Vertical Alignment:** `flex-end` (aligns inputs to bottom).
- **Labels:** Uppercase, `800` weight, `10px` font size, `height: 2.2em`.

---

## Output Table Alignment

To maintain visual symmetry, the data output must align with the top card.

- **Table Wrap:** `min-width: 1100px` (matches card width).
- **Behavior:**
    - In **Rebalance** mode, the table naturally fills or exceeds `1100px`.
    - In **Build** mode, the table may be narrower, but its container must remain `1100px` wide.
- **Max Height:** Calculated dynamically to ensure the header remains sticky without overlapping page content.

## Status Bar

- Rendered as `#status` immediately below `.card-info` inside `.form-card`.
- White text, `12px`, `font-weight: 500`.
- `margin: 3px 0 0` — no `min-height`; card does not reserve blank space when status is empty.
- Error state: `color: #f87171; font-weight: 700`.

## Collapsible Groups

### Expand / Collapse All Buttons

Two secondary buttons — **Expand All** and **Collapse All** — are placed in `#table-controls`, a flex strip (`display:flex; gap:6px; padding:6px 16px 2px`) positioned directly above the output table divs (`#output`, `#build-output`), outside the form card. `#table-controls` is hidden (`display:none`) until a result is available, then shown as `display:flex`.

Styling: same secondary-button class as Export CSV buttons (`outline` variant or equivalent). Labels: `Expand All` / `Collapse All`.

### Group Header Row Visual Style

| Property | Value |
|---|---|
| Background | `rgba(255,255,255,0.06)` — slightly lighter than data rows |
| Font weight | `700` |
| Color | `#fff` |
| Cursor | `pointer` |
| Toggle glyph | `▼` (expanded) or `▶` (collapsed); positioned before the funded year integer with `margin-right: 6px` |
| Bracket year | Append `*` to the funded year integer; header row also gets `background: rgba(173,216,230,0.15)` (matches light-blue bracket theme, but muted for the header) |

### Child Row Visibility

When a group is collapsed (`data-expanded="false"`):
- All `.fy-child` rows with matching `[data-fy]` receive the `hidden` attribute.
- All `.acct-subrow` rows with matching `[data-fy]` receive the `hidden` attribute.
- The group header row itself remains visible at all times.

When expanded:
- `hidden` is removed from all matching `.fy-child` and `.acct-subrow` rows.
- Excess sub-rows (`.excess-subrow.fy-child`) are revealed along with the main TIPS rows — they are inseparable children of their group.
- Individual per-TIPS account expansion state (set via `style.display`) is preserved: rows that were individually collapsed before the group was collapsed remain individually hidden after expansion.

### Default State

All groups render expanded on initial table output. `data-expanded="true"` is set on every `fy-group-header` row; no `hidden` attributes are added to child rows.

### Totals Row

The tfoot totals row is always visible regardless of group expand/collapse state. Its values are unchanged — they reflect the full ladder totals including excess (as before).
