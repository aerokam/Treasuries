# 2.2 Yield Curves Specification

## Purpose
Visualizes the shift and twist in the yield curves over the active time range.

## Snapshots
Two datasets are plotted for each curve (TIPS and Nominal):
- **Start of Range**: Dashed blue line — every maturity read on the **start date**.
- **Latest / End**: Solid red line — every maturity read on the **end date**.

The start and end **dates** are decided once, from the **10Y TIPS** (`US10YTIPS`) — the single reliable feed — and shared across all three curves (TIPS, Nominal, BEI). The sparse, irregular **5Y TIPS feed must never drive date decisions** (see [`1.0_Operation.md`](./1.0_Operation.md) symbol-reliability note). Because `fetchOne()` already clamps `rangeData['US10YTIPS']` to the active window, its first/last points *are* the start/end dates, and they automatically snap past any non-trading day the user picks.

Each maturity is then read **on those exact dates** via `valueOnDate(sym, dateStr, pickLast)`:
- Start uses the day's first print (the close, for a historical day); End uses the last print on the end day.
- A maturity with **no print on that date is left as a gap** (`null`) — datasets use `spanGaps: false`, so the curve breaks at that maturity rather than borrowing a value from an adjacent date. This is why a flaky 5Y TIPS shows a gap instead of pulling in an out-of-window reading.

## Labeling
Both datasets are labeled with the ET timestamp of the **10Y TIPS** point that fixed the date (e.g., `06/15 15:00 ET` for the ~3 PM daily close; `04/08 09:30 ET` for intraday). The label is identical across all three curves.

## Custom Date Range
When the **Custom** range is active (see 2.1):
- **Start** = the picked start date. If that day is a non-trading day (no 10Y TIPS print), it snaps **forward** to the next trading day; the label reflects the actual date used.
- **End** = the picked end date, snapping **backward** the same way.
- Snapping is a side effect of anchoring to the window-clamped 10Y TIPS series — no separate calendar logic.

The live-tip merge in `fetchOne()` also enforces the start-date floor on appended intraday points, so a symbol whose stored history has no point inside the window (again, typically 5Y TIPS) cannot leak pre-start bars into `rangeData[sym]`.
