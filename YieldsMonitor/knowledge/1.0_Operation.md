# 1.0 Yields Monitor Overview

## Purpose
The Yields Monitor provides real-time and historical visualization of Treasury (Nominal) and TIPS (Real) yields. It is designed for intraday precision and long-term trend analysis.

## Data Architecture

**2D, 10D ranges**: Fetch intraday data from CNBC GraphQL API (`1D` and `5D` timeRange).

**1Y, 2Y, 3Y, 10Y, ALL ranges**: 
- Fetch historical baseline from R2 (`Treasuries/yields-history/history.json`, a single nested-by-symbol file) — daily 3PM closes refreshed by updateYieldsHistory.js
- Append latest intraday yields from CNBC `5D` feed to ensure "Latest Yield" and "Day Change" reflect current market data

**If R2 fetch fails**: Returns null; app displays "No data available." To fix: ensure R2 CORS headers are set to allow browser requests, or implement a backend proxy.

**No local fallback code**: App fetches only from remote sources (CNBC, R2).

### R2 stores (under `Treasuries/yields-history/`)

| Store | Key pattern | Written by | Purpose |
|---|---|---|---|
| **Daily-close baseline** | `history.json` (nested by symbol) | `updateYieldsHistory.js` | Long-range/curve baseline — one daily 3PM close per symbol, all 14 symbols in one file |
| **Intraday-raw audit archive** | `intraday-raw/{symbol}/{YYYYMMDD}.json` | `archiveIntraday.js` | Immutable daily snapshot of the raw `1D`+`5D` feeds per symbol, for offline inspection of any past close window (Goal A audit) |
| **Close-timing probe log** | `close-probe/{symbol}.csv` | `probeClose.js` | Temporary investigation log of the last `1D` bar over time, to pin down when the 17:05 print posts |

### Scheduled jobs (Windows Task Scheduler — `scripts/setup-windows-tasks.ps1`)

| Task | When (ET) | Script | Role |
|---|---|---|---|
| `YieldsHistory` | weekdays 17:00 | `updateYieldsHistory.js` | Refresh consolidated daily-close baseline |
| `IntradayArchive` | weekdays 17:05 | `archiveIntraday.js` | Capture raw 1D+5D feeds (all 14 symbols) |
| `CloseProbe` | weekdays 17:05, +15min ×1h | `probeClose.js` | Log US10YTIPS last 1D bar to find reliable 17:05 post time |

### Symbol reliability

The **US5YTIPS (5Y TIPS) feed is flaky** — sparse, irregular prints. Do **not** depend on it for close timing or as a reference symbol. Use **US10YTIPS (10Y TIPS)** as the reliable reference. The archive still captures all 14 symbols; this caveat governs which symbol any *logic* should anchor to.

## Tab Architecture
The monitor is organized into three analysis views:
- **Time Series**: Individual maturity charts with intraday shading and historical trending. (See 2.1)
- **Yield Curves**: Snapshot comparison of curves at the start vs. end of the range. (See 2.2)
- **Breakeven Inflation**: Visualization of market-implied inflation expectations. (See 2.3)

## Symbol List — Yield & Change Display

The sidebar symbol list shows the **latest yield and day change for every available security**, regardless of whether it is selected (checked) or not. Data is fetched for all symbols on every update; the yield/change display reflects that data for all entries. Only the charting area is gated on selection.

## Yield Change Calculation

The "day change" shown next to each symbol (e.g. `−0.016%`) is calculated as:

```
change = latest_yield − closeP.yield
```

**`closeP` selection** — walks backward through `calculationData` (prefers `_5D` cache over `_1D`) to find the last bar from the previous trading day where the bar timestamp is at **exactly 17:00 ET** or earlier (condition: `hour < 17 || (hour === 17 && minute === 0)`). This anchors the reference to the official end-of-trading-day close.

The tooltip shows: `Since MM/DD/YYYY, 17:00 ET close (X.XXX%)`.

### Deviation from CNBC

CNBC's displayed yield change uses a different reference: the `previous_day_closing` field from their Tradeweb quote service. This value reflects an **earlier settlement quote** — apparently a mid-afternoon official price (~14:00–15:51 ET range on observed days) — rather than the 5 PM trading close.

**Example (2026-04-08, US1Y):**
| Source | Reference time | Reference yield | Change vs. 3.677% |
|---|---|---|---|
| This app | 17:00 ET | 3.693% | −0.016% |
| CNBC | ~15:00 ET (Tradeweb settlement) | 3.685% | −0.008% |

The discrepancy (~0.008%) is expected and intentional. Our 17:00 reference is the actual trading close; CNBC's is an earlier official settlement publication (likely Tradeweb's intraday closing quote, distinct from end-of-day trading).

## Intraday Feed Structure & the 17:05 Consolidated Close

The CNBC `1D` and `5D` chart feeds are **24-hour continuous** (electronic/overnight trading), but each weekday has a clear day-session structure. Charts shade 08:00–17:00 ET as market-open; outside that is after-hours. Empirically (verified across all 14 symbols, every weekday in the `intraday-raw` archive):

1. **Continuous day-session trading** runs to roughly 16:56–17:04 ET (cadence is liquidity-driven; illiquid symbols print sparsely).
2. **A single consolidated close print is stamped at exactly `17:05` ET.** This is a **fixed-time end-of-day consolidation, not a random last trade** — it appears for every symbol every weekday regardless of liquidity, even when continuous trading stopped minutes earlier, and even after a `17:04` bar for liquid symbols. Drift from the last continuous bar is sub-bp (0–0.5bp), i.e. a settle value, not a new move.
3. **A ~2.5-hour session break follows** (17:05 → ~19:30 ET), after which the overnight electronic session resumes.

### The 17:05 print is the session close — but NOT CNBC's *daily* close

> **Critical:** the `17:05` print is the actual end-of-**session** trading close, but it is **not** the value CNBC stores as the **daily** close. CNBC's daily close (what the history files are built from) is the **~3 PM benchmark** — see *Daily-close basis* below and the full analysis in **[`Close_Price_Investigation.md`](./Close_Price_Investigation.md)**.

The `1D` feed is authoritative for the 17:05 session close:

| Feed | Resolution | 17:05 representation | Authoritative? |
|---|---|---|---|
| `1D` | 1-min | clean `17:05` consolidated stamp | **Yes** — use this |
| `5D` | 5-min | close bucketed into grid at ~17:02–17:04; no clean 17:05 | Fallback only (if `1D` unavailable) |

**Evidence (06/09/2026, raw 1D bars):**
```
US10Y     17:02=4.5200  17:04=4.5160  17:05=4.5200  →[gap]→ 19:30
US5YTIPS  16:56=1.7960               17:05=1.7990  →[gap]→ 19:30
US1YTIPS  17:01=1.0860               17:05=1.0990  →[gap]→ 19:30
US1M      17:01=3.6600               17:05=3.6600  →[gap]→ 19:30
```

### Data provenance (Tradeweb)

CNBC's Treasury/TIPS yield data is sourced from **Tradeweb** (attributed on the CNBC quote pages). Tradeweb, with FTSE Russell, also runs a formal **Tradeweb FTSE U.S. Treasury Benchmark Closing Prices** product (launched 2024-06-10; FTSE Russell = administrator, Tradeweb = calculation agent; IOSCO-compliant). That benchmark snapshots at **3:00pm and 4:00pm ET**; a **2025-10-17 methodology change** emphasizes the **4:00pm ET** "risk-clearing" level. The benchmark is a **paid LSEG/Tradeweb data product** — it is *not* available in the free CNBC chart feed this app uses.

**Three distinct reference times** (do not conflate):

| Time | What it is | Where it appears |
|---|---|---|
| **3pm / 4pm ET** | Tradeweb **FTSE benchmark closing price** (official, IOSCO; 4pm after 2025-10-17) | CNBC quote-service `previous_day_closing` field (the "~15:00 ET Tradeweb settlement" in *Deviation from CNBC* above) |
| **~17:00 ET** | last *continuous-trading* chart bar | chart intraday stream |
| **17:05 ET** | end-of-session **consolidation print** at the SIFMA 5:00pm ET cash close (+5min; early-close days 2:00pm ET) | the actual session close; shown in 2D/10D views — **not** the persisted daily baseline |

### Daily-close basis: the ~3 PM benchmark (NOT 17:05)

Empirically, **CNBC's stored daily close equals the ~3:00 PM ET mid-afternoon benchmark, not the 17:05 print** (verified across US10Y, US10YTIPS, US30YTIPS — daily close matches the 15:00 intraday value within 0–2bp and runs ~1–2bp below 17:05). This aligns with the U.S. Treasury's official **CMT (nominal) and real/TIPS** yield curves, which use **FRBNY indicative quotations at ~3:30 PM ET**, and with the Tradeweb FTSE benchmark (3pm/4pm). The mid-afternoon snapshot is the industry "official close" convention. Full evidence: **[`Close_Price_Investigation.md`](./Close_Price_Investigation.md)**.

**Decision** (feed resolution is daily for 1Y/2Y/3Y, **weekly** at 10Y, **quarterly** at ALL — so only 10Y/ALL need a persistent store):

| Scope | Close basis | Mechanism |
|---|---|---|
| **1Y / 2Y / 3Y** | ~3 PM benchmark | **Reread fresh** from CNBC daily feeds each run — identical to CNBC's window (no drift), no store. |
| **10Y / ALL** | ~3 PM benchmark | **Persistent merge store**: coarse deep history + accumulated **daily** recent closes (finer than CNBC's weekly/quarterly). Persist **completed days only** (run-timing non-critical). |
| **2D / 10D** | actual **17:05** session close (tentative) | already renders real intraday bars incl. the 17:05 print. |

The 17:05 basis is infeasible for history (the `1D` feed spans only ~2 days). The `IntradayArchive` / `CloseProbe` jobs remain **informational** — they record where the session actually ends; they are not the source for the persisted daily baseline. Full analysis & resolution table: **[`Close_Price_Investigation.md`](./Close_Price_Investigation.md)**.

**Open — day-change anchor:** the sidebar `closeP` uses `≤17:00` (picks ~16:59). Decide whether short-view day-change references the 17:05 session close or the 3 PM basis, to match whichever close each view adopts.

## Shared Interaction & Navigation
- **Timezone**: All displays are in **Eastern Time (ET)**.
- **Precision**: Yields are displayed with 3 decimal places (e.g., `4.321%`).
- **Panning & Zooming**: 
    - Mouse Wheel: XY zoom.
    - Ctrl+Scroll: X-axis zoom only.
    - Shift+Scroll: Y-axis zoom only.
    - Keyboard (Arrows, +/-): Panning and zooming around view center.
- **Y-Axis Auto-Rescale**: Automatic rescaling to visible data bounds ensures maximum vertical resolution during navigation.
- **Sync Zoom & Pan**: When enabled (default), zooming or panning any time series chart automatically synchronizes the X-axis (time) of all other active charts. This allows for easy comparison of yield movements across different maturities. Y-axis scaling remains independent per chart to maintain optimal resolution for each symbol's yield range.
