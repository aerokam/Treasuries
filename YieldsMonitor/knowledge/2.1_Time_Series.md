# 2.1 Time Series Specification

## Layout
Charts are grouped into **TIPS Yields** and **Treasury Yields** rows, sorted by maturity.

## Intraday Visual Standards (2D / 10D Ranges)
- **AH Shading**: After-hours (pre-08:00 and post-17:00 ET) are shaded light gray (`rgba(148, 163, 184, 0.13)`).
- **Weekend Shading**: Full Sat/Sun ET blocks are shaded.
- **Market Bounds**: Vertical dashed lines mark exactly 08:00 and 17:00 ET for each trading day.
- **Dynamic Segments**: Chart lines are slightly faded during after-hours/weekends to emphasize non-primary trading sessions.

## X-Axis Tick Formatting

| Visible window | `time.unit` | Format | Example |
|---|---|---|---|
| `2D` range | `hour` (fixed) | `MM/DD HH:mm` | `04/28 14:30` |
| `10D` range | `day` (fixed) | `MMM D` | `Apr 28` |
| Any range, zoomed < 90 days | `day` | `MMM D` | `May 1` |
| Any range, 90 days–18 months visible | `month` | `MMM YYYY` | `May 2025` |
| Any range, > 18 months visible | `year` | `MMM YYYY` | `2025` |

The `time.unit` is updated dynamically on every zoom and pan complete event via `applyXTimeUnit()` (from `shared/src/chart-time-axis.js`). This controls tick *density* (interval) so Chart.js generates an appropriate number of gridlines at every zoom level. The tick label *format* is separately enforced by the axis `callback` using the same 90-day threshold.

`2D` and `10D` ranges keep their units fixed (`hour`/`day`) regardless of zoom — `applyXTimeUnit` is not called for those ranges.

## Sidebar Metrics
- **Latest Yield**: The absolute most recent data point from the CNBC feed.
- **Yield Change**: Difference between Latest Yield and the **Previous Close**.
- **Previous Close Identification**:
    1. Scan backward from current moment.
    2. Identify the first day *prior* to the latest point's day.
    3. Look for a point between **17:00 and 17:05 ET** (market close) on that prior day.
    4. Fall back to the last available point for that prior day if close is missing.
